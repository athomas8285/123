## 第一章：项目概述

### 1.1 这是什么

一个**世界杯竞彩足球预测系统的管理后台网页**。从竞彩网 API 抓取比赛数据，通过蒙特卡洛泊松分布模拟（λ→MC→DDI→评分→评级）生成预测方向，按比赛日生成竞彩 2 串 1 计划方案。

### 1.2 线上地址

| 环境 | 地址 | 启动方式 |
|------|------|---------|
| 本地 | http://127.0.0.1:5021 | `python dashboard.py` |
| 生产 | http://8.136.46.240 | Gunicorn (端口 80) |

### 1.3 技术栈

```
Flask 3.x (Python 3.10) + SQLite (WAL 模式) + 原生 HTML/JS (SPA)
Gunicorn 26.x → 生产环境 | Lucide Icons (CDN) → 图标
NumPy → 蒙特卡洛模拟 | requests → HTTP 请求
```

### 1.4 核心架构

```
浏览器 → Flask dashboard.py → routes/ 7个模块 → SQLite framework.db ← 竞彩网 API
                                                                      ← 500.com
浏览器 → static/panels/*.js (8个面板, ES Module)
静态文件 → data/*.json (18个数据文件)
```

```javascript
  btn.addEventListener("click", () => {
    // 移除所有 active class
    document.querySelectorAll(".nav-item").forEach(b => b.classList.remove("active"));

  });
});

// 面板加载器（带缓存防止重复）
const panelLoaded = {};
window.panelLoaded = panelLoaded;  // 暴露给子模块失效缓存用

const loaders = {
  overview,            // overview.js export
  console: loadConsole,
  matches: loadMatches,
  fundamental: loadFundamental,
  plans: loadPlans,
  featured: function(){ /* 刷新 iframe src */ },
  review: loadReview,
  system: loadSystem
};

async function loadPanel(name) {
  if (panelLoaded[name]) return;
  panelLoaded[name] = true;
  if (loaders[name]) await loaders[name]();
}
```

| lambda_h_final / lambda_a_final | REAL | λ 最终值 |
| lambda_diff | REAL | λ差值 |
| physical_home_win/draw/away_win | REAL | 物理概率 |
| market_home_win/draw/away_win | REAL | 市场概率 |
| ddi_home_win/draw/away_win | REAL | DDI |

| s7_reason / trap_analysis / key_risk | TEXT | AI 分析 |
| diagnosis | TEXT | 诊断文本 |
| run_id | INTEGER | FK → runs.id |

### 4.3 关键查询模式

**跨 run 去重（取每个 match_id 最新一条）：**
```sql
SELECT * FROM matches
WHERE id IN (SELECT MAX(id) FROM matches GROUP BY match_id)
```

**优先取有 direction 的 row（如果有预测过的历史 row，优先用那个）：**
```sql
SELECT * FROM matches m1 WHERE m1.id = (
    SELECT id FROM matches m2 WHERE m2.match_id = m1.match_id
    ORDER BY CASE WHEN m2.direction IS NOT NULL AND m2.direction != '' THEN 0 ELSE 1 END, m2.id DESC LIMIT 1
)
```

### 5.2 数据查询 API

#### /api/dashboard/overview (GET)

**源码位置：** `routes/overview_routes.py` → `def overview()`

**返回结构：**
```json
{
  "stats": {"total": 104, "scored": 98, "predicted": 100, "hit": 52, "miss": 46, "hitrate": 53.1},
  "today_matches": [{match_id:"周二095", home:"阿根廷", away:"埃及", direction:"胜", ...}],
  "missing_results": 2,
  "health": {"sp_missing": 19, "total": 104},
  "flow_steps": [
    {"id":"fetch", "label":"获取竞彩网比赛", "done":false, "desc":"未获取", "action":true},
    {"id":"data", "label":"获取比赛数据", "done":false, "desc":"待采集 3 场", "action":true},
    {"id":"predict", "label":"运行预测", "done":false, "desc":"待预测 3 场", "action":true},
    {"id":"results", "label":"查询赛果", "done":false, "desc":"需补 2 场", "action":true},
    {"id":"plan", "label":"生成计划池", "done":false, "desc":"待生成", "action":true}
  ],
  "plan_info": {"plan_count": 45}
}
```

**数据来源：**
- `total`: `COUNT(*) FROM matches WHERE id IN (SELECT MAX(id) FROM matches GROUP BY match_id)` — 去重后的 match_id 数
- `scored`: 有实际赛果的比赛数，从 `_RESULT_CACHE`（跨 run 赛果缓存）计算
- `predicted`: `direction IS NOT NULL AND direction != ''` 的比赛数
- `hit / miss`: 从 `_RESULT_CACHE` 统计 `hit==1` / `hit==0`
- `today_matches`: 有方向但无赛果的比赛，`match_time[:10] == today.strftime("%Y-%m-%d")`
- `missing_results`: 有方向但无赛果 = `predicted - scored`
- `flow_steps.done`: 判断当天是否有比赛、是否已采集、是否已预测、是否有计划
- `plan_info.plan_count`: 读 `plan_data.json` 里 `date_groups` 的 `plan_2` + `plan_3` 方案总数

### 5.3 操作端点

#### /api/dashboard/action/fetch_jczq (GET)

**流程：**
1. requests → `https://webapi.sporttery.cn/gateway/uniform/football/getMatchCalculatorV1.qry?channel=c&poolCode=hhad,had,ttg,crs,hafu`
2. 带完整浏览器 headers（`Accept-Language: zh-CN` 关键）
3. 如果 403 → Playwright fallback（Chromium headless，但服务器一般不可用）
4. 保存 `raw_jczq.json` → 统计新比赛 → 返回 `{ok, new_matches, new_count}`

#### /api/dashboard/action/confirm_jczq_matches (POST)

**请求体：** `{match_ids: ["周二095","周二096"]}`

**流程：**
1. 从 `raw_jczq.json` 读取对应的比赛详情
2. 插入 DB：`INSERT INTO matches (run_id, match_id, home, away, match_time, event, league, match_date, jc_sp_home/draw/away, jc_handicap, jc_hhad_win/draw/lose, hafu_odds, ttg_odds, crs_odds)`
3. 同步 `match_info.json` 和 `locked_data.json`

#### /api/dashboard/action/fetch_results (GET)

**流程：**
1. 查 DB 最近 5 天内已完赛但无赛果的比赛
2. 对每个日期调 500.com → 解析 HTML 表格 → 提取 `cells[6]` 获取比分（格式 `(0:1) 3:2`）→ 正则提取 `(\d+):(\d+)` × 2
3. 写入 DB actual_score + half_full → 自动计算 hit → 返回弹窗表格数据

#### /api/dashboard/action/run_predict (GET/POST)

**请求参数：** `?match_ids=周二095,周二096` 或 POST body `{match_ids: [...]}`

**流程：**
1. 后台线程启动 `python run_all.py --match-ids 周二095,周二096`
2. 标准输出重定向到 `data/pipeline_run.log`
3. 返回 `{ok, msg}`，前端开始每 3 秒轮询 `/api/dashboard/pipeline_progress`

#### /api/dashboard/action/gen_plan (GET/POST)

**流程：**
1. `python gen_plan.py --predicted` → plan_data.json
2. `python gen_plan_html.py` → static/plan.html
3. `python gen_featured.py` → featured_data.json
4. `python gen_featured_html.py` → static/featured.html

```
└──────────┴──────────┴──────────┴──────────┘
```

**卡片数字来源：**
- `s.total = len(all_match_rows)` — 去重后的比赛总数（`GROUP BY match_id` 后 count）
- `s.scored = COUNT(mid WHERE mid IN _RESULT_CACHE)` — 有赛果的比赛
- `s.predicted = COUNT(mid WHERE direction != '')` — 有预测方向的比赛
- `s.hit = COUNT(mid WHERE _RESULT_CACHE[mid].hit == 1)`
- `s.miss = COUNT(mid WHERE _RESULT_CACHE[mid].hit == 0)`
- `s.hitrate = round(hit/(hit+miss)*100, 1) %`
- `today_matches = COUNT(mid WHERE direction != '' AND actual_score == '' AND match_time[:10] == today)`

**点击行为：** 每张卡片的 `data-filter` 属性决定跳转目标，统一调用 `navigateToMatches(filter)` → 切换到比赛面板 → 设置对应筛选条件。

| 卡片 | data-filter | 跳转到比赛面板筛选 |
|------|-----------|-----------------|
| 总比赛 | all | "全部" |
| 已有赛果 | done | "已赛果" |
| 已预测 | pred | "预测中" |
| 命中 | hit | "命中" |
| 未命中 | miss | "未命中" |
| 今日比赛 | today | "今日" |

## 第十四章：gen_plan.py — 计划池生成器 (V5.3)

### 14.1 数据输入

- `match_info.json` — SP 赔率 (jc_sp_win/draw/lose / jc_hhad_win/draw/lose), handicap, hafu_odds
- `rating_result.json` — direction, rating, fit_score
- `monte_carlo_result.json` — physical 概率, lambda 值, jc_handicap_prob, top2_hf
- `fundamental_analysis.json` — narrative (基本面信号)
- `raw_jczq.json` — hafu_odds (半全场赔率，用于半全场腿)
- `framework.db` — hit 值缓存

### 14.2 腿生成规则

**方向单腿 (6 个):**
- 从 SP 赔率取: 主胜 (sp_h/物理 HWin)、平局 (sp_d/物理 Draw)、客胜 (sp_a/物理 AWin)
- 从让球赔率取: 让胜 (hhad_win/hcp_prob.rang_sheng)、让平 (hhad_draw/hcp_prob.rang_ping)、让负 (hhad_lose/hcp_prob.rang_fu)
- 每条腿: `{mid, match, date, type:"方向", option, odds, mp, src_mp, fit, rating}`

**方向打包腿 (4 个):**
- 胜/平 打包: mp = 物理 HWin+Draw, 赔率 = 1/(1/sp_h+1/sp_d)
- 负/平 打包: mp = 物理 AWin+Draw, 赔率 = 1/(1/sp_a+1/sp_d)
- 让胜/平 打包: mp = rang_sheng+rang_ping, 赔率 harmonic
- 让平/负 打包: mp = rang_ping+rang_fu, 赔率 harmonic

**半全场单腿 (2 个):**
- 从 MC top2_hf 取 (如 ["平胜", "胜胜"])
- 从 raw_jczq 取对应的 hafu_odds[key], key 映射: "胜胜"→"hh", "平胜"→"dh", "负负"→"aa", "平负"→"da" 等

**半全场打包腿:**
- 如果 top2 是 (胜胜,平胜) → "主不败" 打包
- 如果 top2 是 (负负,平负) → "客不败" 打包

### 14.3 方向兼容矩阵 (_check)

- 非方向腿: 总是通过
- 方向腿: `_direction_compatible(pred, hcp, opt)`:
  - 平/让平: 总是兼容
  - pred ∈ {胜,让胜} + hcp<0: 排除 opt ∈ {负,客胜}
  - pred ∈ {胜,让胜} + hcp≥0: 排除 opt ∈ {负,客胜,让负}
  - pred ∈ {负,让负} + hcp>0: 排除 opt ∈ {胜,主胜}
  - pred ∈ {负,让负} + hcp≤0: 排除 opt ∈ {胜,主胜,让胜}
  - pred=平: 总是兼容

### 14.4 显示层 handicap 替换 (|hcp|=1 时)

- hcp=-1: 方向打包(让胜/平) → 用方向(主胜)的 `display_option` 和 `display_odds` (当主胜赔率更高时)
- hcp=+1: 方向打包(让平/负) → 用方向(客胜)的 `display_option` 和 `display_odds` (当客胜赔率更高时)
- 仅显示层替换，不影响组合生成和排序

### 14.5 组合生成 (2 腿交叉)

```python
for l1 in legs_for_date:
    for l2 in legs_for_date:
        if l1["mid"] == l2["mid"]: continue  # 必须不同比赛
        # 计算组合赔率
        opt_odds = l1["odds"] × l2["odds"]
        # EV = type_hitrate × avg_payout
        type_hitrate = (1/l1.odds) × (1/l2.odds)
        ev = type_hitrate × (min_ret + max_ret) / 2
        min_fit = min(l1.fit, l2.fit)
```

### 14.6 排序

```python
combos.sort(key=lambda c: (abs(c["raw_opt_odds"] - 2.0), c["min_odds"], -c["ev"]))
```

1. 赔率最接近 2.0
2. 保底回报最高
3. EV 最高

### 14.7 分档

- 2.0 档: opt_odds ∈ [1.9, 排大 2.5], 逐步放宽区间直到有方案, TOP 5
- 3.0 档: opt_odds ∈ (2.5, 5.0), TOP 5

## 第十五章：gen_featured.py — 精选计划单生成器

### 15.1 策略

```python
STRATEGY = {'base_stake': 1000, 'multipliers': [1,1,2,4,8,16], 'max_rounds': 6}
```

### 15.2 每日精选

从计划池的 2.0 档取第一名 (plan_2[0]) 作为当日 pick。

### 15.3 判定逻辑 (judge_leg_hit)

**方向腿:**
- "主胜"/"胜" → `hg > ag`
- "客胜"/"负" → `hg < ag`
- "平局"/"平" → `hg == ag`
- "让胜" → `hg + hcp - ag > 0`
- "让平" → `hg + hcp - ag == 0`
- "让负" → `hg + hcp - ag < 0`

**方向打包腿:** 拆开 options，任一条命中即算命中

**半全场腿:** `half_full == opt`

**半全场打包腿:**
- "主不败" → `half_full ∈ ("胜胜","平胜")`
- "客不败" → `half_full ∈ ("负负","平负")`

### 15.4 追回逻辑

- 命中 → 轮次归零, profit = stake×(min_odds-1), cumulative_profit += profit
- 偏离 → 轮次+1, profit = -stake, cumulative_profit += profit
- pending (no actual_score) → 不计盈亏

## 第二十章：编码注意事项

1. **所有文件 UTF-8** — 中文比较字符串必须正确编码，否则 hit 计算失败
2. **PowerShell 管道吞中文** — 必须写独立 .py 文件执行，禁止 `@"...""@` 方式
3. **Python → python3 软链接** — 服务器 `ln -sf /usr/bin/python3 /usr/bin/python`
4. **panelLoaded 缓存** — 操作后需 `panelLoaded.xxx = false` 强制重新加载
5. **iframe 缓存破坏** — 刷新时加 `?_t=Date.now()` 参数
6. **控制台 `_renderPreview` 每次重建 HTML** — 按钮事件绑定需在渲染后重新执行
7. **竞彩网 API 仅限国内 IP** — 海外服务器无法访问，需在国内服务器或本地操作
